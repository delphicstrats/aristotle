## Slide 8: SCI Linkages

### Title
SCI Institutionalizes Governance

### Heading
SCI creates interdependent intelligence records that link all aspects of managing the strategy-to-stakeholder-to-outcomes chain.

### Body
Display a graphical representation of the six intelligence record types with flow arrows:
1. Priority flows to Objective
2. Action flows to Objective
3. Stakeholder flows to Action
4. Action and Stakeholder flow to Talking Point Framework
5. Talking Point Framework flows to Activity
6. Activity flows to Stakeholder

Below the graphical representation breakout the core function of each intelligence record type:
1. Priority: An enduring strategic goal
2. Objective: A discrete, time-bound goal that advances a Priority
3. Action: An act that a Stakeholder must take to achieve an Objective
3. Stakeholder: The independent external actor
4. Talking Point Framework: A set of information and messaging points to change a Stakeholder's posture
5. Activity: A communications, engagement or operational action that impacts a Stakeholder

## Slide 11: AI

### Title
AI-Powered Intelligence

### Heading
SCI leverages AI to create audit-ready intelligence record evidence, increase analytical capability and reduce user cognitive burden.

### Body
Display graphical representations of the following three capabilities:
1. Human-in-the-Loop Evidence Validation:
  - External APIs monitor sources including news, social media and organizational websites for content related to existing intelligence records.
  - User-uploaded files are similarly assessed.
  - Evidence records are created when content related to existing intelligence records is found.
  - Evidence records are run through an LLM/ML pipeline that pulls exact quotations from evidence records, storing them as evidence citations.
  - Evidence citations for any given intelligence record are sent to the Primary and all Delegate Owners of the record.
  - Owners review the evidence citations, and if they agree it supports their record then they validate the citation.
2. AI Assist:
  - Users interact with their assigned intelligence records primarily through entry forms.
  - Key fields in each entry form that draw from existing context are given an AI Assist button.
  - The AI Assist button when clicked provides a value generated by a subject matter expert agent that assesses all evidence and associated record context.
  - Users can accept the recommendation in whole, edit it as preferred or reject it.
3. SCI Assistant:
  - The SCI application features an always-on chatbot.
  - The chatbot operates in sessions that each user can initiate to accomplish specific tasks.
  - All sessions are saved and can be easily reviewed and referenced in new session.
  - The SCI Assistant has access to all records across the platform and can execute complex queries leveraging individual SME agents.
  - The SCI Assistant can create custom reports based on these queries, or make updates to intelligence records to which the user is assigned.

## Slide 11: Complete Solution

### Title
The Solution is SCI

### Heading
SCI provides the governance of your strategy-to-stakeholder-to-outcomes pipeline.

### Body
The problem SCI addresses is not new, and has been articulated in Resource Dependence Theory and Department of Defense Strategic Communications doctrine.
Until now, however, instituting such a governance framework has been too difficult.
AI provides the necessary, always-on analytical capability and reduces the significant user load inherent in a platform that distributes ownership across the organization.
SCI can help your organization better understand and align your stakeholders so that you can achieve better strategic outcomes.
